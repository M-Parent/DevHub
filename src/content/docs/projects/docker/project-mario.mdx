## Prerequisites

- Docker installed and running
- Web browser
- Port 8600 available (or choose another)

```bash
# Verify Docker is running
docker --version
docker ps
```

---

## Quick Start

### One Command Deployment

```bash
docker run -d -p 8600:8080 pengbai/docker-supermario
```

That's it! Open `http://localhost:8600` in your browser.

---

## Step-by-Step Deployment

<Steps>

1. **Pull the image**

   ```bash
   docker pull pengbai/docker-supermario
   ```

   This downloads the Super Mario game container image from Docker Hub.

2. **Run the container**

   ```bash
   docker run -d \
     --name mario \
     -p 8600:8080 \
     pengbai/docker-supermario
   ```

   | Flag           | Description                               |
   | -------------- | ----------------------------------------- |
   | `-d`           | Run in background (detached)              |
   | `--name mario` | Give container a friendly name            |
   | `-p 8600:8080` | Map host port 8600 to container port 8080 |

3. **Verify it's running**

   ```bash
   docker ps
   ```

   ```
   CONTAINER ID   IMAGE                      STATUS         PORTS
   abc123         pengbai/docker-supermario  Up 5 seconds   0.0.0.0:8600->8080/tcp
   ```

4. **Play the game**

   Open your browser to: `http://localhost:8600`

</Steps>

<Aside type="tip" title="Controls">
  Use arrow keys to move, and spacebar or 'S' key to jump/run. Have fun!
</Aside>

## Managing the Container

### View Logs

```bash
docker logs mario
docker logs -f mario  # Follow live
```

### Stop the Game

```bash
docker stop mario
```

### Start Again

```bash
docker start mario
```

### Remove Container

```bash
docker rm -f mario
```

---

## Customizing the Port

If port 8600 is busy, use any available port:

```bash
# Use port 3000
docker run -d --name mario -p 3000:8080 pengbai/docker-supermario

# Access at http://localhost:3000
```

---

## Troubleshooting

### Port Already in Use

```bash
# Find what's using the port
lsof -i :8600

# Use a different port
docker run -d -p 8787:8080 pengbai/docker-supermario
```

### Container Won't Start

```bash
# Check for existing container with same name
docker ps -a | grep mario

# Remove it
docker rm mario

# Try again
docker run -d --name mario -p 8600:8080 pengbai/docker-supermario
```

### Can't Access in Browser

```bash
# Check container is running
docker ps | grep mario

# Check port mapping
docker port mario

# Check logs for errors
docker logs mario
```

## What You Learned

This simple project demonstrated:

1. **`docker pull`** - Download images from Docker Hub
2. **`docker run`** - Create and start containers
3. **`-p` flag** - Port mapping between host and container
4. **`-d` flag** - Running containers in background
5. **`--name`** - Naming containers for easy management
6. **Container lifecycle** - start, stop, rm commands
