NGINX is a powerful web server used by major hosting companies worldwide. It handles high volumes of requests simultaneously, offering faster loading times and better performance than most other web servers.

<Aside type="tip" title="Why NGINX?">
  NGINX uses fewer resources than Apache and is excellent for static content,
  reverse proxying, and load balancing.
</Aside>

## What is NGINX?

| Feature         | Description                                |
| --------------- | ------------------------------------------ |
| **Type**        | Web server, reverse proxy, load balancer   |
| **Performance** | High concurrency, low memory usage         |
| **Config**      | Simple, declarative configuration files    |
| **Use Cases**   | Static sites, API gateway, SSL termination |

## Install NGINX

<Steps>

1. **Update package repositories**

   ```bash
   sudo apt update
   ```

2. **Install NGINX**

   ```bash
   sudo apt install nginx
   ```

3. **Start and enable NGINX**

   ```bash
   # Start NGINX now
   sudo systemctl start nginx

   # Enable NGINX to start on boot
   sudo systemctl enable nginx
   ```

4. **Verify installation**

   ```bash
   curl localhost
   ```

   You should see the default NGINX welcome page HTML.

</Steps>

### Configure Virtual Host

<Steps>

1. **Create site configuration**

   ```bash
   sudo tee /etc/nginx/sites-available/devhub-site.conf << 'EOF'
   server {
       listen 80;
       listen [::]:80;

       server_name devhub.local;

       root /var/www/devhub-site;
       index index.html;

       location / {
           try_files $uri $uri/ =404;
       }
   }
   EOF
   ```

2. **Enable the site**

   Create a symbolic link to enable the site:

   ```bash
   sudo ln -s /etc/nginx/sites-available/devhub-site.conf /etc/nginx/sites-enabled/
   ```

3. **Test configuration**

   Always test before restarting:

   ```bash
   sudo nginx -t
   ```

   Expected output:

   ```
   nginx: configuration file /etc/nginx/nginx.conf syntax is ok
   nginx: configuration file /etc/nginx/nginx.conf test is successful
   ```

4. **Restart NGINX**

   ```bash
   sudo systemctl restart nginx
   ```

5. **Test your site**

   ```bash
   curl localhost
   ```

</Steps>

## Configuration Explained

```nginx
server {
    listen 80;              # Listen on port 80 (HTTP)
    listen [::]:80;         # Listen on IPv6 port 80

    server_name devhub.local;  # Domain name

    root /var/www/devhub-site; # Document root
    index index.html;          # Default file

    location / {
        try_files $uri $uri/ =404;  # Try file, then dir, then 404
    }
}
```

| Directive     | Purpose                            |
| ------------- | ---------------------------------- |
| `listen`      | Port to listen on                  |
| `server_name` | Domain/hostname for this site      |
| `root`        | Directory containing website files |
| `index`       | Default file to serve              |
| `try_files`   | Order to try serving files         |

## Useful NGINX Commands

```bash
# Check status
sudo systemctl status nginx

# Test configuration syntax
sudo nginx -t

# Reload config (no downtime)
sudo systemctl reload nginx

# Restart completely
sudo systemctl restart nginx

# View error logs
sudo tail -f /var/log/nginx/error.log

# View access logs
sudo tail -f /var/log/nginx/access.log
```
