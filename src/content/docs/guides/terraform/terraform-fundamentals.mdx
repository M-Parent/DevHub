Terraform is an open-source Infrastructure as Code (IaC) tool by HashiCorp. It allows you to define and provision infrastructure using a declarative configuration language called HCL (HashiCorp Configuration Language).

<Aside type="tip" title="Why Terraform?">
  Terraform lets you manage infrastructure across multiple cloud providers (AWS,
  Azure, GCP) and on-premises environments using a single, consistent workflow.
</Aside>

---

## Prerequisites

Before starting, ensure you have:

- Ubuntu Linux (20.04 or later)
- `sudo` privileges
- Git installed
- An AWS account (for cloud examples)

---

## Installing Terraform on Ubuntu

<Steps>

1. **Install required packages**

   ```bash
   sudo apt-get update && sudo apt-get install -y gnupg software-properties-common
   ```

2. **Add HashiCorp GPG key**

   ```bash
   wget -O- https://apt.releases.hashicorp.com/gpg | \
   gpg --dearmor | \
   sudo tee /usr/share/keyrings/hashicorp-archive-keyring.gpg > /dev/null
   ```

3. **Verify the key fingerprint**

   ```bash
   gpg --no-default-keyring \
   --keyring /usr/share/keyrings/hashicorp-archive-keyring.gpg \
   --fingerprint
   ```

4. **Add HashiCorp repository**

   ```bash
   echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(grep -oP '(?<=UBUNTU_CODENAME=).*' /etc/os-release || lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
   ```

5. **Install Terraform**

   ```bash
   sudo apt update
   sudo apt-get install terraform
   ```

6. **Verify installation**

   ```bash
   terraform version
   ```

</Steps>

## Setting Up Your Terraform Project

### Create a Git Repository

It's best practice to version control your Terraform code. This ensures you can track changes and recover from mistakes.

<Steps>

1. **Create a project directory**

   ```bash
   mkdir terraform-project
   cd terraform-project
   ```

2. **Initialize Git**

   ```bash
   git init
   ```

3. **Create a .gitignore file**

   ```bash
   cat > .gitignore << 'EOF'
   # Terraform
   .terraform/
   *.tfstate
   *.tfstate.*
   *.tfvars
   !example.tfvars
   crash.log
   override.tf
   override.tf.json
   *_override.tf
   *_override.tf.json
   .terraformrc
   terraform.rc
   EOF
   ```

4. **Connect to remote repository**

   ```bash
   git add .
   git commit -m "Initial commit"
   git remote add origin <your-repository-url>
   git push -u origin main
   ```

</Steps>

<Aside type="caution" title="Never Commit Sensitive Data">
  Your `.tfvars` files often contain sensitive data like passwords and API keys.
  Always add them to `.gitignore` and use environment variables or secret
  management tools in production.
</Aside>

## Basic Terraform Commands

| Command              | Description                                      |
| -------------------- | ------------------------------------------------ |
| `terraform init`     | Initialize working directory, download providers |
| `terraform plan`     | Preview changes before applying                  |
| `terraform apply`    | Create or update infrastructure                  |
| `terraform destroy`  | Remove all managed infrastructure                |
| `terraform fmt`      | Format configuration files                       |
| `terraform validate` | Validate configuration syntax                    |

## Your First Configuration

Create a simple `main.tf` file:

```hcl
# Configure the AWS Provider
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}

provider "aws" {
  region = "ca-central-1"
}

# Create a VPC
resource "aws_vpc" "main" {
  cidr_block = "10.0.0.0/16"

  tags = {
    Name = "devhub-vpc"
  }
}
```

Run the workflow:

```bash
# Initialize providers
terraform init

# Preview changes
terraform plan

# Apply changes
terraform apply
```
